1. Какая технология называется межсетевым обменом (Интернетworking)?

Межсетевой обмен (Internetworking) — это технология объединения в единую сеть нескольких локальных и территориальных сетей, обеспечивающая взаимодействие между ними посредством стандартных протоколов. В контексте TCP/IP, межсетевой обмен подразумевает передачу пакетов через сети, использующие различные технологии передачи данных, как локальные, так и глобальные сети​.

2. Объясните понятие «протоколы» в контексте технологий обмена данными. Что они включают? Приведите примеры.

Протоколы — это наборы правил и инструкций, определяющих порядок взаимодействия устройств в сети и обмена данными между ними. Протоколы включают правила маршрутизации, обмена данными, управление соединениями и обеспечение безопасности. Примеры протоколов: TCP (Transmission Control Protocol) для надежной передачи данных, UDP (User Datagram Protocol) для более быстрой, но ненадежной передачи, HTTP (Hypertext Transfer Protocol) для передачи гипертекста в сети.

3. Назовите отличия TCP/IP от других средств передачи данных.

TCP/IP отличается от других средств передачи данных своей универсальностью, популярностью и масштабируемостью. Он поддерживает множество типов сетевых архитектур и предоставляет протоколы для передачи данных на различных уровнях, от физического до прикладного. В отличие от некоторых других протоколов, таких как IPX/SPX, TCP/IP стандартизирован для работы в глобальных сетях и является основой Интернета​.

4. Дайте определение понятию «сокет».

Сокет — это конечная точка двустороннего обмена данными между приложениями, работающими в сети. Сокет используется для установления связи, передачи данных и завершения соединения между клиентом и сервером.

5. Опишите функцию, которая используется для приема данных через сокет потока (протокол TCP).

Для приема данных через сокет потока в протоколе TCP используется функция recv(). Она принимает следующие параметры: сокет (s), указатель на буфер для приема данных (buf), длина буфера (len), и флаги (flags). Функция возвращает количество принятых байт или ошибку в случае неудачи​.

6. Назовите функцию, используемую для создания сокета. Опишите ее параметры.

Функция для создания сокета — socket(domain, type, protocol). Параметры:
domain — семейство протоколов (например, AF_INET для Интернета);
type — тип сокета (например, SOCK_STREAM для TCP);
protocol — протокол, который будет использоваться (обычно 0 для автоматического выбора)​(Komlichenko_komp).

7. Опишите функцию, которая используется для пересылки данных через сокет потока (протокол TCP).
Для пересылки данных через сокет потока используется функция send(). Она принимает: сокет (s), указатель на буфер с данными (buf), длину данных для отправки (len) и флаги (flags). Возвращает количество успешно отправленных байт или ошибку в случае неудачи​(Komlichenko_komp).

8. Что возвращает функция accept() в том случае, если соединение открывается успешно?

Функция accept() возвращает новый дескриптор сокета, который будет использоваться для управления новым соединением с клиентом. Если соединение не удалось, возвращается код ошибки​(Komlichenko_komp).

9. Назовите функцию, которая используется в приложении-клиенте для посылки запроса на открытие соединения. Опишите ее параметры.

Функция connect(s, addr, addrlen) используется в клиенте для посылки запроса на открытие соединения. Параметры:
s — сокет;
addr — структура с адресом и портом, к которому подключается клиент;
addrlen — размер структуры адреса. Эта функция возвращает 0 в случае успеха и ошибку в противном случае​(Komlichenko_komp).
